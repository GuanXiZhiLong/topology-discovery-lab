# 开发计划

## 第一阶段：项目骨架

目标：建立可被 Codex 稳定理解和迭代的项目结构。

任务：

1. 创建项目目录。
2. 创建 `README.md`。
3. 创建 `AGENTS.md`。
4. 创建 `docs/` 文档结构。
5. 创建 `pyproject.toml`。
6. 创建 `.python-version`。
7. 创建 `.gitignore`。
8. 创建 PR 模板。
9. 创建基础 Python 包目录。
10. 创建空测试目录。

验收标准：

1. 项目结构与架构文档一致。
2. Python 包可以被导入。
3. `pytest` 可以运行。
4. `ruff` 可以运行。
5. `mypy` 可以运行。
6. Codex 可以根据 `AGENTS.md` 理解项目边界。

## 第二阶段：配置系统

目标：实现 YAML 配置读取和校验。

任务：

1. 定义配置模型。
2. 实现 `load_config(path: str)`。
3. 支持扫描目标、SNMP、SSH、Neo4j 配置。
4. 编写配置文件样例。
5. 编写正常和错误配置测试。

验收标准：

1. 可以读取 `config/config.example.yaml`。
2. 缺少必填字段时给出明确错误。
3. 配置类型错误时给出明确错误。
4. 不允许提交真实密码。
5. 测试覆盖正常配置和异常配置。

## 第三阶段：数据模型

目标：定义统一拓扑数据模型。

任务：

1. 定义 `AliveHost`。
2. 定义 `DeviceNode`。
3. 定义 `InterfaceNode`。
4. 定义 `LinkEdge`。
5. 定义 `TopologySnapshot`。
6. 定义 `DiscoveryError`。
7. 编写模型校验和非法数据测试。

验收标准：

1. 模型字段清晰。
2. 必填字段受到校验。
3. 非法 IP、非法状态、非法置信度可以被拦截。
4. 所有写入 Neo4j 的数据都能用模型表达。
5. 测试覆盖主要模型。

## 第四阶段：ICMP 存活探测

目标：实现指定网段内设备存活发现。

任务：

1. 支持 CIDR 网段展开。
2. 支持单个 IP 扫描。
3. 支持 ICMP 探测。
4. 支持超时和重试配置。
5. 支持并发限制。
6. 返回 `AliveHost` 列表。
7. 编写 mock 和异常测试。

验收标准：

1. 输入网段可以生成 IP 列表。
2. 存活设备返回 `reachable=true`。
3. 不可达设备不导致整体崩溃。
4. 单个 IP 探测失败不会中断整个扫描。
5. 所有探测都有超时。
6. 测试不依赖真实网络。

## 第五阶段：SNMP 基础采集

目标：对存活设备进行 SNMP 基础信息采集。

任务：

1. 实现 SNMP get 和 walk。
2. 获取 `sysName`、`sysDescr`、`sysObjectID`。
3. 获取接口列表、接口状态、接口 MAC。
4. 将结果转化为结构化模型。
5. 编写 SNMP mock 测试。

验收标准：

1. SNMP 成功时返回结构化结果。
2. SNMP 失败时返回错误信息。
3. SNMP 超时时不会阻塞整体流程。
4. 不支持 SNMP 的设备不会导致整体崩溃。
5. 接口数据可以被解析为 `InterfaceNode` 所需字段。

## 第六阶段：拓扑解析

目标：将采集结果转化为统一拓扑快照。

任务：

1. 将 `AliveHost` 转化为候选设备。
2. 将 SNMP 信息转化为 `DeviceNode`。
3. 将 SNMP 接口信息转化为 `InterfaceNode`。
4. 实现设备、接口和基础链路去重。
5. 生成 `TopologySnapshot`。
6. 编写解析、重复数据和异常输入测试。

验收标准：

1. 相同设备不会重复生成节点。
2. 相同接口不会重复生成节点。
3. 无法识别的设备类型标记为 `unknown`。
4. 部分采集失败不影响其他设备进入快照。
5. 生成的 `TopologySnapshot` 可以通过模型校验。

## 第七阶段：Neo4j 写入

目标：将拓扑快照写入 Neo4j。

任务：

1. 实现 Neo4j 连接管理。
2. 实现设备节点、接口节点写入。
3. 实现 `HAS_INTERFACE` 和 `CONNECTED_TO` 关系写入。
4. 使用 `MERGE` 避免重复写入。
5. 支持 `last_seen` 更新。
6. 编写 repository 和 Cypher 参数测试。

验收标准：

1. 设备节点不会重复。
2. 接口节点不会重复。
3. 设备和接口关系可以正确建立。
4. 链路关系可以正确建立。
5. Neo4j 连接失败时有明确错误。
6. 不在日志或异常中暴露密码。

## 第八阶段：最小端到端流程

目标：实现一次完整扫描和写入流程。

任务：

1. 在 `main.py` 中串联配置读取。
2. 执行 ICMP 扫描。
3. 执行 SNMP 采集。
4. 构建拓扑快照。
5. 写入 Neo4j。
6. 输出运行统计。
7. 编写端到端 mock 测试。

验收标准：

1. 一条命令可以启动发现流程。
2. 失败设备不会导致整体流程失败。
3. 成功设备可以写入 Neo4j。
4. 输出统计清晰。
5. 端到端测试可以运行。

## 第九阶段：LLDP/CDP 链路发现

目标：扩展链路发现能力。

任务：

1. 研究 LLDP 相关 SNMP OID。
2. 采集 LLDP 邻居信息。
3. 解析本端接口、远端设备和远端接口。
4. 生成 `LinkEdge`。
5. 写入接口级 `CONNECTED_TO` 关系。
6. 编写样例数据测试。

验收标准：

1. 可以从 LLDP 数据生成链路。
2. 链路不会重复。
3. A 到 B 和 B 到 A 不会重复写两条。
4. 链路具备置信度。
5. 链路具备发现方式。

## 第十阶段：增量更新

目标：支持重复扫描后的拓扑变化维护。

任务：

1. 支持 `last_seen` 更新。
2. 支持设备、接口、链路状态更新。
3. 支持发现任务记录。
4. 支持离线设备标记。
5. 支持过期链路标记。

验收标准：

1. 重复扫描不会重复创建节点。
2. 设备再次发现会更新 `last_seen`。
3. 未再次发现的设备可以标记为 `offline`。
4. 未再次发现的链路可以标记为 `stale`。
5. 可以查询最近一次拓扑状态。

## 当前阶段最终验收标准

项目整体合格需要满足：

1. 项目结构完整。
2. 文档体系完整。
3. Python 3.11 配置明确。
4. 依赖边界明确。
5. 当前阶段范围明确。
6. 暂缓内容明确。
7. 数据模型清晰。
8. Neo4j 图模型清晰。
9. SNMP、SSH、Neo4j 安全规则明确。
10. 测试策略明确。
11. Codex 开发和 Review 流程明确。
12. 下一步任务明确。
