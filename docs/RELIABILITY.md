# 可靠性设计

## 可靠性目标

当前阶段可靠性目标：

1. 单个设备失败不能导致整个扫描失败。
2. 单个协议失败不能导致整个设备完全失败。
3. 外部连接必须有超时。
4. 错误必须结构化记录。
5. 数据库写入失败必须明确暴露。
6. 敏感信息不能出现在错误信息中。
7. 重复执行任务不会破坏已有数据。
8. 部分成功结果可以正常写入。

## 网络层失败

网络层失败包括：

- 目标 IP 不可达。
- ICMP 被防火墙拦截。
- 网络延迟过高。
- 路由不可达。
- DNS 解析失败。
- 本机权限不足导致无法发送探测包。

处理原则：

1. 标记目标不可达。
2. 记录错误原因。
3. 继续扫描其他目标。
4. 不抛出导致整体中断的异常。

## 超时策略

建议默认值：

```yaml
scan:
  timeout_seconds: 2
  retry_count: 1

snmp:
  timeout_seconds: 2
  retry_count: 1

ssh:
  timeout_seconds: 5
```

原则：

1. 不让单个 IP 长时间阻塞。
2. 大网段扫描必须支持并发限制。
3. 失败结果应返回结构化信息。
4. 所有外部操作必须支持超时。

## 重试策略

当前阶段只实现简单重试，不实现复杂退避算法。

允许：

- ICMP 简单重试。
- SNMP 简单重试。
- Neo4j 连接失败后有限重试。

暂不实现：

- 指数退避。
- 分布式重试队列。
- 死信队列。
- 自动任务恢复系统。

## 不应重试的情况

- 配置文件格式错误。
- IP 地址格式非法。
- SNMP community 明显为空。
- SSH 用户名为空且 SSH 已启用。
- Neo4j 密码为空。
- 数据模型校验失败。

## 可以重试的情况

- ICMP 临时超时。
- SNMP 临时超时。
- SSH 连接超时。
- Neo4j 临时连接失败。

## 失败隔离

设备级隔离：

1. 一个设备失败，不影响其他设备。
2. 所有失败项进入 `DiscoveryError`。
3. 最终快照可以包含部分成功结果。

协议级隔离：

1. ICMP 成功但 SNMP 失败时，仍可保留基础候选设备。
2. SNMP 成功但 SSH 失败时，仍可生成设备和接口。
3. SSH 默认关闭，不应影响主流程。

写入级隔离：

1. Neo4j 写入失败必须明确报告。
2. 当前阶段可以整体写入失败，不需要复杂部分提交恢复。
3. 未来可扩展为分批写入和失败恢复。

## 幂等要求

Neo4j 写入必须尽量幂等：

1. 设备使用 `MERGE (d:Device {device_id: $device_id})`。
2. 接口使用 `MERGE (i:Interface {interface_id: $interface_id})`。
3. 链路使用稳定 `link_id`。
4. 重复扫描更新属性，不重复创建。
5. 使用 `last_seen` 表示最近发现时间。
